# Результаты тестирования производительности (Вариант 2)

В данном отчете представлено сравнение скорости работы сервисов на **Go (Gin)** и **Python (FastAPI/Uvicorn)** на основе реальных замеров утилитой Apache Benchmark.

## Параметры тестирования
* **Инструмент:** Apache Benchmark (ab)
* **Количество запросов (-n):** 1000
* **Уровень параллелизма (-c):** 10
* **Тестовые эндпоинты:** * `http://localhost:8080/ping` (Go-service)
    * `http://localhost:8000/` (Python-service)

## Сводная таблица результатов

| Метрика | Go (Gin) | Python (FastAPI) |
| :--- | :--- | :--- |
| **Requests per second (RPS)** | **543.09 #/sec** | 303.00 #/sec |
| **Time per request (mean)** | **18.413 ms** | 33.004 ms |
| **Total time for tests** | **1.841 seconds** | 3.300 seconds |
| **99% Percentile** | **52 ms** | 84 ms |
| **Longest request** | **61 ms** | 90 ms |

## Анализ результатов
На основе полученных данных можно сделать следующие выводы:
1. **Производительность:** Сервис на языке **Go** обрабатывает в **1.79 раза больше** запросов в секунду (RPS), чем сервис на Python.
2. **Задержки:** Среднее время ответа (Latency) у Go почти в **1.8 раза ниже**, что обеспечивает более быстрый отклик приложения.
3. **Стабильность:** Анализ перцентилей показывает, что 99% всех запросов в Go укладываются в 52 мс, в то время как у Python этот показатель составляет 84 мс. Это говорит о более высокой стабильности работы Go-сервиса под нагрузкой.

**Вывод:** Тестирование подтверждает более высокую эффективность Go-сервиса в условиях конкурентной нагрузки для **Варианта 2**. При этом оба фреймворка продемонстрировали отличную надежность, успешно обработав 100% запросов без единой ошибки (`Failed requests: 0`).
