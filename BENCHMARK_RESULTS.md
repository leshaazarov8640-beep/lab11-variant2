# Отчет по лабораторной работе №11 (Вариант 2)
**Тема:** Контейнеризация мультиязычных приложений и анализ производительности.

## 1. Выполнение практических заданий
В рамках варианта №2 были выполнены следующие задачи:
* **Задание 2:** Разработан Dockerfile для Go-приложения с использованием многоэтапной сборки (multi-stage build).
* **Задание 4:** Сборка образов и сравнительный анализ их размеров.
* **Задание 6:** Настройка сетевого взаимодействия между контейнерами в среде Docker Compose.

## 2. Сравнение размеров Docker-образов (Задание 4)
На основе данных команды `docker images` проведен анализ занимаемого дискового пространства:

| Репозиторий | Content Size | Особенности сборки |
| :--- | :--- | :--- |
| **lab11-variant2-go-service** | **10 MB** | Multi-stage build (статическая компиляция) |
| **lab11-variant2-python-service** | **51.8 MB** | Базовый образ python-slim |

**Вывод:** Применение многоэтапной сборки для Go позволило минимизировать размер образа, оставив в финальном слое только исполняемый файл. Образ Python оказался в 5 раз тяжелее из-за необходимости наличия интерпретатора и библиотек.

## 4. Сетевая связность (Задание 6)
Контейнеры объединены в общую сеть через `docker-compose`. Это позволяет сервисам взаимодействовать по внутренним именам хостов. Проверка доступности подтверждена успешным прохождением тестов Apache Benchmark.

## 5. Общий вывод
В ходе работы успешно настроена контейнеризация для двух разных технологических стеков. Сервис на **Go** показал лучшие результаты как по скорости обработки запросов (быстрее в 1.8 раза), так и по компактности Docker-образа (меньше в 5 раз). Оба сервиса демонстрируют стабильную работу в изолированной среде.
